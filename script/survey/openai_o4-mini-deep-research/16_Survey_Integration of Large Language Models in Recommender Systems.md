# 引言  
近年来大规模预训练语言模型（LLM）在自然语言处理领域取得了突破性成果，引发了推荐系统领域对其应用的浓厚兴趣 ([arxiv.org](https://arxiv.org/abs/2308.10837#:~:text=In%20this%20paper%2C%20we%20propose,trained%20language%20models)) ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=Recommender%20systems%20have%20seen%20significant,challenges%20such%20as%20scalability%20issues))。研究者们尝试利用 LLM 强大的语义理解能力为推荐系统赋能，以应对传统基于 ID 的协同过滤方法在语义表示不足、冷启动等问题上的局限性 ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=Recommender%20systems%20have%20seen%20significant,challenges%20such%20as%20scalability%20issues)) ([arxiv.org](https://arxiv.org/abs/2412.16933#:~:text=This%20paper%20explores%20the%20use,tuning%20framework%20that%20refines%20a))。然而，如何架构性地将 LLM 的语言语义与推荐系统的协同语义有效结合仍是一大挑战。当前代表性工作集中在设计新的预训练或微调框架、采用生成式推荐策略、以及通过向量量化和提示技术桥接语义差距等方向展开创新。下面按照方法类别对这些工作进行分类综述，并分析其核心思路和实验结论。  

## 方法分类与代表作  

### 预训练模型与表示学习  
**RecSysLLM (Chu et al., 2023)** 研究者针对将 LLM 通用推理与推荐域知识融合的问题，提出了预训练的推荐系统模型 RecSysLLM ([arxiv.org](https://arxiv.org/abs/2308.10837#:~:text=based%20on%20LLMs,developing%20unified%20recommendation%20systems%20by))。该方法通过独特设计的训练数据、训练目标和推理过程，使模型在保留 LLM 原有语言理解能力的同时，注入领域协同信号 ([arxiv.org](https://arxiv.org/abs/2308.10837#:~:text=based%20on%20LLMs,developing%20unified%20recommendation%20systems%20by))。实验结果表明，RecSysLLM 在多个基准和真实场景数据上均明显提高了推荐性能，展示出利用预训练语言模型构建统一推荐系统的可行性 ([arxiv.org](https://arxiv.org/abs/2308.10837#:~:text=based%20on%20LLMs,developing%20unified%20recommendation%20systems%20by))。  
**RLMRec (Ren et al., 2023)** 提出了一种模型无关的框架 RLMRec，用以增强现有推荐器的表示学习能力 ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=address%20these%20challenges%2C%20we%20propose,view)) ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=alignment%20framework,textual%20signals%20through%20mutual%20information))。该框架利用 LLM 对用户和物品的文本描述进行编码，结合辅助文本信号构建用户/物品画像，并通过交叉视图对齐任务（基于互信息最大化）将 LLM 的语义空间与协同空间对齐 ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=address%20these%20challenges%2C%20we%20propose,view)) ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=alignment%20framework,textual%20signals%20through%20mutual%20information))。理论分析和实验验证表明，RLMRec 的互信息对齐策略能够显著提升表示质量，并在集成到多种主流推荐模型后表现出良好的稳定性和对噪声信息的鲁棒性 ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=address%20these%20challenges%2C%20we%20propose,view)) ([arxiv.org](https://arxiv.org/abs/2310.15950#:~:text=alignment%20framework,textual%20signals%20through%20mutual%20information))。  

### 序列推荐  
**Harte et al. (2023)** 探讨了在序列推荐场景中应用 LLM 的多种方式 ([arxiv.org](https://arxiv.org/abs/2309.09261#:~:text=datasets%20show%20that%20initializing%20the,our%20experiments%20to%20ensure%20reproducibility))。他们设计了三种不同方法来利用 LLM：例如，将预训练的 LLM 文本嵌入用于初始化最先进的序列模型（如 BERT4Rec），显著优化了推荐结果；另外也尝试直接使用 LLM 生成的语义嵌入来进行检索式推荐。实验表明，将 LLM 嵌入向量注入 BERT4Rec 中可带来 15–20% 的 NDCG 提升，且采用 LLM 嵌入直接进行检索也能产生与传统方法相当的效果 ([arxiv.org](https://arxiv.org/abs/2309.09261#:~:text=datasets%20show%20that%20initializing%20the,our%20experiments%20to%20ensure%20reproducibility))。该工作强调了 LLM 在捕获用户行为序列中长期语义关联的潜力。  
**E4SRec (Li et al., 2023)** 面向序列推荐提出了一种高效扩展的解决方案 ([arxiv.org](https://arxiv.org/abs/2312.02443#:~:text=traditional%20recommender%20systems%20that%20exclusively,com%2FHestiaSky%2FE4SRec))。该方法输入用户的 ID 序列作为模型输入，保证输出结果限定于候选列表内；通过一次前向传播直接生成完整的推荐排名列表，并仅学习极少量的可插拔参数，而保持大规模 LLM 参数冻结不变 ([arxiv.org](https://arxiv.org/abs/2312.02443#:~:text=traditional%20recommender%20systems%20that%20exclusively,com%2FHestiaSky%2FE4SRec))。实验证明，E4SRec 在四个真实序列推荐数据集上兼具有效性、效率和扩展性，在生成多个推荐结果的同时显著降低了计算成本 ([arxiv.org](https://arxiv.org/abs/2312.02443#:~:text=traditional%20recommender%20systems%20that%20exclusively,com%2FHestiaSky%2FE4SRec))。  
**RSLLM (Zheng et al., 2024)** 将用户行为序列视作一种新的“语言”，提出了 RSLLM 框架 ([arxiv.org](https://arxiv.org/abs/2412.16933#:~:text=This%20paper%20explores%20the%20use,tuning%20framework%20that%20refines%20a))。该方法结合传统序列推荐和 LLM 双重优势，将用户/物品的 ID 嵌入与文本特征通过投影器对齐，并采用两阶段微调策略：先使用纯文本提示进行预微调，再针对目标域进行统一提示微调，使模型逐步学习并融合行为序列中的协同信息 ([arxiv.org](https://arxiv.org/abs/2412.16933#:~:text=This%20paper%20explores%20the%20use,tuning%20framework%20that%20refines%20a))。实验证明，RSLLM 能有效引入序列推荐知识到 LLM 中，在测试中验证了方法的有效性。  

### 生成式推荐  
**GPT4Rec (Li et al., 2023)** 受启发于搜索引擎机制，将推荐问题转化为生成检索任务 ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=address%20these%20limitations%2C%20we%20present,BM25%20search%20engine%2C%20our%20framework)) ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))。首先对用户交互历史中的物品标题生成假设性的“查询”语句，然后使用 BM25 等检索算法以这些查询检索推荐物品。该框架利用 GPT-2 生成多查询（beam search）来捕获用户多方面兴趣，生成的查询不仅可解释用户意图，也能推荐冷启动物品 ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=address%20these%20limitations%2C%20we%20present,BM25%20search%20engine%2C%20our%20framework)) ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))。实验结果显示，相较于最先进的推荐方法，GPT4Rec 在两个公共数据集上的 Recall@K 分别提升了约 75.7% 和 22.2% ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))；同时，多查询生成策略显著提高了推荐结果的多样性和对用户多兴趣覆盖度 ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))。  
**GenRec (Ji et al., 2023)** 提出直接使用 LLM 生成推荐物品的生成式推荐框架 ([arxiv.org](https://arxiv.org/abs/2307.00457#:~:text=one%20as%20in%20traditional%20discriminative,based%20generative%20recommendation%20in)) ([arxiv.org](https://arxiv.org/abs/2307.00457#:~:text=underscores%20the%20potential%20of%20LLM,better%20results%20on%20large%20dataset))。该方法设计了专门的提示语，使得 LLaMA 模型能将用户-物品交互文本作为上下文，直接“输出”需要推荐的目标物品，而不是对每个候选项进行打分。研究者对 LLaMA 进行了提示增强和基于用户-物品文本交互数据的微调，以学习用户偏好和物品特征 ([arxiv.org](https://arxiv.org/abs/2307.00457#:~:text=one%20as%20in%20traditional%20discriminative,based%20generative%20recommendation%20in))。大规模实验证明，GenRec 在多个基准数据集上取得了显著优于传统推荐基线的效果 ([arxiv.org](https://arxiv.org/abs/2307.00457#:~:text=underscores%20the%20potential%20of%20LLM,better%20results%20on%20large%20dataset))，表明基于 LLM 的生成式方法在推荐场景具有潜力。  
**个性化叙事生成 (Sayana et al., 2024)** 探索了在对话推荐中使用 LLM 生成丰富的用户叙事 ([arxiv.org](https://arxiv.org/abs/2410.16780#:~:text=understanding%20recommender%20signals%20and%20generating,CF%20and%20content%20embeddings%20contribute))。该工作发布了名为 REGEN 的数据集，其中扩充了亚马逊商品评论数据集，包含用户对推荐结果的解释性叙述（例如偏好说明、推荐理由、历史总结等）。基于此数据，提出了将协同过滤（CF）嵌入与 LLM 结合的融合架构作为基线 ([arxiv.org](https://arxiv.org/abs/2410.16780#:~:text=metrics%2C%20and%20perform%20an%20automated,We))。实验结果表明，同时使用用户的协同过滤特征和内容特征输入到 LLM 可以显著提高生成质量，通过融合不同类型的嵌入，生成文本指标（BLEU、Rouge 等）平均提升了约 4–12% ([arxiv.org](https://arxiv.org/abs/2410.16780#:~:text=understanding%20recommender%20signals%20and%20generating,CF%20and%20content%20embeddings%20contribute))，验证了 LLM 在理解推荐信号并生成个性化解释方面的能力。  

### 协同语义融合模型  
**LC-Rec (Zheng et al., 2023)** 针对 LLM 语言语义与推荐系统协同语义之间的鸿沟，提出了 LC-Rec 模型 ([openreview.net](https://openreview.net/forum?id=bI999sgLuN#:~:text=Abstract%3A%20Recently%2C%20large%20language%20models,which%20can%20better%20integrate%20language)) ([openreview.net](https://openreview.net/forum?id=bI999sgLuN#:~:text=For%20item%20indexing%2C%20we%20design,characterized%20by%20the%20learned%20item))。该模型通过学习型向量量化为物品分配有意义的索引（item indices），并设计一系列特殊微调任务来深度融合语言与协同语义 ([openreview.net](https://openreview.net/forum?id=bI999sgLuN#:~:text=Abstract%3A%20Recently%2C%20large%20language%20models,which%20can%20better%20integrate%20language)) ([openreview.net](https://openreview.net/forum?id=bI999sgLuN#:~:text=For%20item%20indexing%2C%20we%20design,characterized%20by%20the%20learned%20item))。在 LC-Rec 中，物品索引与文本表征相互映射，训练过程中显式地将生成式语言模型与推荐任务对齐。实验结果表明，LC-Rec 在多个数据集上均显著优于传统推荐模型和其他 LLM-based 方法，验证了其通过对齐策略桥接两类语义的有效性 ([openreview.net](https://openreview.net/forum?id=bI999sgLuN#:~:text=indices,based%20recommenders.%20Our%20code%20is))。  
**CLLM4Rec (Zhu et al., 2023)** 提出了 CLLM4Rec，首个紧密结合 LLM 模式和 ID 模式的生成式推荐系统 ([arxiv.org](https://arxiv.org/abs/2311.01343#:~:text=simultaneously,related))。该方法通过将用户/物品 ID 词元扩展到预训练 LLM 的词表，并采用“软+硬”提示策略对用户/物品的协同和内容语义进行联合学习 ([arxiv.org](https://arxiv.org/abs/2311.01343#:~:text=simultaneously,related))。此外，引入互正则化机制以抑制噪声信息对训练的干扰 ([arxiv.org](https://arxiv.org/abs/2311.01343#:~:text=simultaneously,related)) ([arxiv.org](https://arxiv.org/abs/2311.01343#:~:text=information%20from%20noisy%20user%2Fitem%20content,com%2Fyaochenzhu%2Fllm4rec))。在微调过程中，CLLM4Rec 加入了预测头，使得模型可以基于历史交互生成多个推荐物品，且无需预先候选列表 ([arxiv.org](https://arxiv.org/abs/2311.01343#:~:text=information%20from%20noisy%20user%2Fitem%20content,com%2Fyaochenzhu%2Fllm4rec))。实验证明该模型能够同时建模协同和语义信息，为多项推荐提供有效支持。  
**A-LLMRec (Kim et al., 2024)** 研究者观察到传统协同过滤（CF）模型在暖启动时性能优异，而 LLM 对冷启动有优势，因而提出了 A-LLMRec 框架 ([arxiv.org](https://arxiv.org/abs/2404.11343#:~:text=collaborative%20knowledge,world%20datasets))。该方法将预训练的先进 CF 模型产生的用户/物品嵌入作为额外输入，使 LLM 可以直接利用协同知识做推荐 ([arxiv.org](https://arxiv.org/abs/2404.11343#:~:text=collaborative%20knowledge,world%20datasets))。A-LLMRec 具有模型无关性（可适配多种 CF 模型）和高效性（减少了对 LLM 的大规模微调需求）两大优势 ([arxiv.org](https://arxiv.org/abs/2404.11343#:~:text=high,the%20understanding%20of%20the%20collaborative))。实验证明，在冷/热启动、少样本甚至跨域场景下，A-LLMRec 均优于多种对比方法 ([arxiv.org](https://arxiv.org/abs/2404.11343#:~:text=high,the%20understanding%20of%20the%20collaborative))，表明通过联合利用 CF 嵌入与 LLM 可实现更稳健的推荐性能。  
**EAGER-LLM (Hong et al., 2025)** 提出了一种将外源行为信号与语义信息无侵入式融合到 LLM 推荐框架中的方法 ([arxiv.org](https://arxiv.org/abs/2502.14735#:~:text=non,LLM%27s%20effectiveness%20through%20rigorous%20testing))。该方法首先使用“双源知识索引”（将行为信号编码为连续索引序列）来丰富物品表示，其次设计多尺度对齐重构任务，促进模型深入理解协同与内容信号 ([arxiv.org](https://arxiv.org/abs/2502.14735#:~:text=non,LLM%27s%20effectiveness%20through%20rigorous%20testing))。此外，引入退火式适配器，在保持语言理解能力的同时微调推荐性能 ([arxiv.org](https://arxiv.org/abs/2502.14735#:~:text=non,LLM%27s%20effectiveness%20through%20rigorous%20testing))。在三个公共数据集上的严格测试表明，EAGER-LLM 在综合考虑协同和文本信息方面效果优异，显著提升了推荐准确性 ([arxiv.org](https://arxiv.org/abs/2502.14735#:~:text=non,LLM%27s%20effectiveness%20through%20rigorous%20testing))。  

## 实验与评价总结  
总体来看，结合大规模语言模型的推荐方法普遍能够在多样化的场景中提升推荐准确度和多样性。多项实验表明，将 LLM 生成或嵌入的语义信息融入推荐框架后，冷启动问题得到缓解，常见指标（如 NDCG、Recall）出现显著提升 ([arxiv.org](https://arxiv.org/abs/2309.09261#:~:text=datasets%20show%20that%20initializing%20the,our%20experiments%20to%20ensure%20reproducibility)) ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))。例如 Harte 等的工作中，将 LLM 嵌入用于 BERT4Rec，可带来 15–20% 的 NDCG 提升 ([arxiv.org](https://arxiv.org/abs/2309.09261#:~:text=datasets%20show%20that%20initializing%20the,our%20experiments%20to%20ensure%20reproducibility))；GPT4Rec 中引入检索式生成后，Recall@K 提升高达 75% 以上 ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies))。此外，生成式方法通常能增加推荐结果的多样性和可解释性，例如多查询生成策略和叙事生成都提升了结果的覆盖率和文档质量 ([arxiv.org](https://arxiv.org/abs/2304.03879#:~:text=items.%20With%20GPT,discussed%20with%20qualitative%20case%20studies)) ([arxiv.org](https://arxiv.org/abs/2410.16780#:~:text=understanding%20recommender%20signals%20and%20generating,CF%20and%20content%20embeddings%20contribute))。在效率方面，不同工作提出了冻结部分模型参数、仅训练轻量模块或代理器等手段来减轻 LLM 的计算成本 ([arxiv.org](https://arxiv.org/abs/2312.02443#:~:text=traditional%20recommender%20systems%20that%20exclusively,com%2FHestiaSky%2FE4SRec))。综上，现有研究一致表明，在合理设计下，LLM 的引入对推荐系统性能具有积极影响，但同时也带来需要仔细解决的计算和泛化挑战。  

## 趋势与挑战  
展望未来，基于 LLM 的推荐系统研究有若干可能的发展趋势：  
- **深度融合协同过滤与语言模型**：越来越多的工作将探索如何联合预训练的协同过滤模型与 LLM，充分利用两者的优势。比如 A-LLMRec等研究已经表明，结合协同嵌入可以显著提升模型在多种场景（冷启动、跨域等）下的表现 ([arxiv.org](https://arxiv.org/abs/2404.11343#:~:text=high,the%20understanding%20of%20the%20collaborative))。未来将进一步研究多任务联合训练、跨域共享知识等策略，使模型既具备丰富的语义理解能力，又能保有协同信息。  
- **检索增强的生成推荐（RAG）**：引入检索机制来补充 LLM 提示内容成为趋势。诸如 ARAG 等多智能体检索生成框架将用户长期和会话行为通过专业化代理（用户理解、逻辑推理、排序等）分工处理，以更好地捕获动态偏好。该方向能够更灵活地整合外部知识和推荐结果，同时可生成更具上下文相关性的推荐解释。  
- **多模态与知识融合**：未来研究可能加强对多模态信息（如图像、音频、知识图谱等）的融合，使推荐模型能够利用更多维度的上下文。大型语言模型与视觉模型或图神经网络的结合，将进一步提高推荐的准确度和用户理解度，满足丰富场景的需求。  
- **对话式与解释性推荐**：随着对用户体验要求的提高，通过 LLM 提供个性化叙事和解释成为研究热点。基于对话的推荐系统和生成式解释（如用户故事、偏好描述）将更为普遍 ([arxiv.org](https://arxiv.org/abs/2410.16780#:~:text=understanding%20recommender%20signals%20and%20generating,CF%20and%20content%20embeddings%20contribute))。这要求模型不仅推荐相关项目，还能生成符合用户意图的自然语言内容，提高推荐系统的可解释性和用户信任度。  
- **效率与可控性**：实际应用中，如何在保证性能的同时降低 LLM 推理成本是重要挑战。未来可能通过模型蒸馏、剪枝、知识适配器等技术，使得 LLM 更轻量化以适配在线系统。此外，可控推荐（控制生成结果的属性或风格）也将是一个重要方向，确保生成内容符合应用规范。  

## 结论  
近年来，研究者已在多条技术路径上尝试将大型语言模型引入推荐系统，包括预训练框架、序列建模、生成式检索以及协同语义融合等。各类工作表明，LLM 能够显著提升推荐效果，特别是在场景丰富度和多样性方面带来新机遇。然而，如何平衡大模型的计算代价与实用性，以及有效对齐语言模型与推荐系统的语义，仍是亟待解决的问题。综上所述，基于 LLM 的推荐研究正处于高速发展阶段，未来有望在模型架构、优化策略和应用层面继续取得突破。
