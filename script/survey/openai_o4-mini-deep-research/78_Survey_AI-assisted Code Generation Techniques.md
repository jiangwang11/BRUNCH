# 引言  
近年来，随着大规模预训练模型在自然语言处理领域的成功，基于深度学习的代码生成技术迅速兴起。例如，Nijkamp等（2022）提出了**CodeGen**模型，训练了高达16.1B参数的代码语言模型，并采用多轮提示（multi-turn prompt）方式分步生成程序，在HumanEval基准上的零样本性能与当时最好模型相当 ([arxiv.org](https://arxiv.org/abs/2203.13474#:~:text=demonstrating%20that%20it%20is%20competitive,We))。与此同时，大规模多语言代码模型也被提出，如Zheng等（2023）发布的**CodeGeeX**（13B参数、覆盖23种语言）在多语言HumanEval-X数据集上优于同规模模型 ([arxiv.org](https://arxiv.org/abs/2303.17568#:~:text=Large%20pre,Finally%2C%20CodeGeeX%20is%20publicly))。这些工作表明，增大模型规模、采用混合预训练目标及多语种训练等手段可以明显提升代码生成能力。本文对2022–2025年间相关工作进行分类综述，涵盖预训练模型、检索增强、提示工程、规划与迭代生成等多种技术流派。

# 方法分类与代表作  
## 大规模预训练模型  
大规模预训练模型通过在海量代码上学习通用表示来提升代码生成能力。如**CodeGen**采用纯Transformer架构组合自然语言与代码数据进行预训练，并引入多轮编程范式，将单一指令分解为子问题逐步生成目标程序。实验表明，多轮提示的方式能显著提高生成质量 ([arxiv.org](https://arxiv.org/abs/2203.13474#:~:text=demonstrating%20that%20it%20is%20competitive,We))。Guo等（2023）的**LongCoder**关注长上下文代码完成任务，通过滑动窗口自注意力和“桥接桥记忆”机制捕获多文件或长文件内的全局信息，在新的长上下文数据集和CodeXGLUE上均优于此前模型 ([arxiv.org](https://arxiv.org/abs/2306.14893#:~:text=In%20this%20paper%2C%20we%20introduce,definitions%20of%20classes%2C%20functions%2C%20or)) ([arxiv.org](https://arxiv.org/abs/2306.14893#:~:text=structures,com%2Fmicrosoft%2FCodeBERT))。**CodeGeeX**则是一种跨语言的13B代码模型，使用8500亿单词规模的多语言代码语料预训练。在HumanEval-X（包含多种编程语言）测试中，该模型超过同规模多语种代码模型的表现 ([arxiv.org](https://arxiv.org/abs/2303.17568#:~:text=Large%20pre,In))。以上工作均显示：扩展预训练数据和模型规模、设计混合目标（如因果LM、掩码填充、对比学习等）以及指令微调等手段，有助于提升代码生成的泛化与多任务能力。

## 检索增强生成  
检索增强方法引入外部代码库信息，类似开发者查找示例代码的行为，以补充代码语言模型的上下文。Lu等（2022）提出**ReACC**框架，在代码补全任务中结合语义检索模块，通过从外部语料库中找出相似代码片段并结合Copy机制生成后续代码，Stage-wise训练后在CodeXGLUE上达到新的最优水平 ([arxiv.org](https://arxiv.org/abs/2203.07722#:~:text=Code%20completion%2C%20which%20aims%20to,art%20performance%20on%20CodeXGLUE%20benchmark))。Zhang等（2023）的**RepoCoder**针对仓库级代码完成，采用相似性检索与预训练模型迭代生成的流程，能够利用跨文件上下文信息。他们在新构建的RepoEval评测集上发现，此方法相对只使用文件内上下文的基线方案，使代码补全精度提升了10%以上 ([arxiv.org](https://arxiv.org/abs/2303.12570#:~:text=levels%20of%20granularity,com%2Fmicrosoft%2FCodeT%2Ftree%2Fmain%2FRepoCoder)) ([arxiv.org](https://arxiv.org/abs/2303.12570#:~:text=indicate%20that%20RepoCoder%20significantly%20improves,analysis%2C%20providing%20valuable%20insights%20for))。Pham等（2024）提出**RepoHyper**，利用新颖的“仓库语义图”和图扩展/链接预测检索机制，进一步优化仓库级代码补全，也明显优于传统方法 ([arxiv.org](https://arxiv.org/abs/2403.06095#:~:text=Code%20Large%20Language%20Models%20,and%20a%20link%20prediction%20algorithm)) ([arxiv.org](https://arxiv.org/abs/2403.06095#:~:text=applied%20to%20the%20RSG%2C%20enabling,AI4Code%2FRepoHyper))。此外，Li等（2023）的**SkCoder**采用“草图”思想：先检索相关代码片段的关键模式（sketch），再根据需求对其进行编辑。该方法打破单纯复制的局限，在多个数据集上令Pass@1指标相较CodeT5-base提升了约30% ([arxiv.org](https://arxiv.org/abs/2302.06144#:~:text=dataset%20collected%20by%20this%20work,our%20SkCoder%20in%20three%20aspects))。总体来看，引入代码检索和重用能够有效补充局部上下文，大幅提高生成的正确率和代码覆盖面。

## 提示工程与分步规划  
针对提示（prompt）的改进和生成流程拆解是近期热门方向。例如，Li等（2023）的**AceCoder**提出双重提示机制：先让模型输出中间产物（如测试用例或需求分析）以澄清需求，再检索相似示例代码提示具体“如何写”。在HumanEval衍生的多种基准上，这种导引式提示使得不同规模的模型Pass@1大幅提升：在MBPP、MBJP等基准上，相对于最优已有方法分别提高了56.4%、70.7%、88.4% ([arxiv.org](https://arxiv.org/abs/2303.17780#:~:text=Pass%40k,developers%20prefer%20programs%20from%20AceCoder))。Ma等（2023）提出**SeCoT**（语义链式思维），在提示中自动引入代码的静态语义信息（数据流、控制流等），使模型在理解需求到代码的映射时更具语义感知。与普通Flat提示相比，SeCoT在HumanEval、HumanEval-ET和MBPP上实现了新高性能 ([arxiv.org](https://arxiv.org/abs/2310.10698#:~:text=the%20powerful%20GPT,large%20models%20and%20code%20generation))。Jiang等（2023）提出**自规划生成**，模拟人类先分解问题再编程的过程：模型首先通过Few-shot提示生成具体的解题步骤，再逐步生成对应代码。实验表明，与直接生成相比，此策略在Pass@1上可提升25.4%，相较于普通CoT也提升11.9%；且人工评估显示其生成代码在正确性、可读性和健壮性方面都更优 ([arxiv.org](https://arxiv.org/abs/2303.06689#:~:text=in%20the%20implementation%20phase%2C%20the,robustness%2C%20as%20assessed%20by%20humans))。以上工作表明，通过设计辅助思考和渐进式生成，使LLM能够更好地理解复杂需求，生成更符合预期的代码。

## 工具辅助与迭代生成  
除单纯的语言模型生成外，引入外部工具和多阶段流水线也是有效策略。Zhang等（2023）的**ToolCoder**启发于开发者查找API文档的过程：利用ChatGPT生成带有工具调用标注的训练数据，微调模型使其学会在生成时自动调用API搜索工具查询合适的函数。该方法在五个代码生成基准测试上平均提升Pass@1约6.2%，Pass@10约9.6%，即使一个小模型也能达到GPT-3.5的效果 ([arxiv.org](https://arxiv.org/abs/2305.04032#:~:text=generation%20benchmarks%2C%20with%20at%20least,3.5%2C%20highlighting%20the))。DeepMind的**AlphaCode**（2022）则采用生成-筛选的流水线：训练大规模代码LM后，对每个竞赛题目生成上万种候选方案，依靠测试用例筛选有效程序。该系统在Codeforces竞赛中取得了第54.3百分位的成绩 ([arxiv.org](https://arxiv.org/abs/2203.07814#:~:text=we%20introduce%20AlphaCode%2C%20a%20system,an%20extensive%20and%20clean%20competitive))，显示大规模样本生成结合程序测试是一种可行的解决复杂编程任务的思路。Ridnik等（2024）的**AlphaCodium**提出了类似的多阶段流程：结合精心设计的提示和中途测试反馈，将GPT-4在难题上的Pass@5命中率从19%提升到44% ([arxiv.org](https://arxiv.org/abs/2401.08500#:~:text=significantly%20improves%20results,broadly%20applicable%20to%20general%20code))。此外，通过强化学习结合编译器或单元测试反馈（例如**CodeRL** ([arxiv.org](https://arxiv.org/abs/2207.01780#:~:text=provide%20dense%20feedback%20signals%20to,achieves%20new%20SOTA%20results%20on))、**StepCoder** ([arxiv.org](https://arxiv.org/abs/2402.01391#:~:text=APPS%2B%20dataset%20for%20RL%20training%2C,and%20StepCoder%20are%20available%20online))）可以进一步优化模型，已在APPS、MBPP等任务上刷新了结果。这一类方法均强调生成-校验-再生成的迭代闭环思路，有效提升生成代码的正确性与可靠性。

# 实验与评价总结  
上述工作大多使用共同的评测基准（如HumanEval、MBPP、CodeXGLUE、APPS、Codeforce竞赛题等），在Pass@K或准确率等指标下给出了量化对比。整体结论显示：利用检索或外部上下文补充（如RepoCoder、ReACC等）可使代码补全正确率提升约10%以上 ([arxiv.org](https://arxiv.org/abs/2303.12570#:~:text=indicate%20that%20RepoCoder%20significantly%20improves,analysis%2C%20providing%20valuable%20insights%20for))；引入规划或链式思维提示（如自规划、链式思考等）在复杂任务上相对直接生成提升约10%–25% ([arxiv.org](https://arxiv.org/abs/2303.06689#:~:text=in%20the%20implementation%20phase%2C%20the,robustness%2C%20as%20assessed%20by%20humans))；工具调用和测试驱动优化也显著提高了性能，其中ToolCoder平均提升6.2% ([arxiv.org](https://arxiv.org/abs/2305.04032#:~:text=generation%20benchmarks%2C%20with%20at%20least,3.5%2C%20highlighting%20the))，CodeRL等框架则在APPS与MBPP上获得新高 ([arxiv.org](https://arxiv.org/abs/2207.01780#:~:text=provide%20dense%20feedback%20signals%20to,achieves%20new%20SOTA%20results%20on)) ([arxiv.org](https://arxiv.org/abs/2402.01391#:~:text=APPS%2B%20dataset%20for%20RL%20training%2C,and%20StepCoder%20are%20available%20online))。此外，多语种和多任务评测表明，专门针对代码预训练的模型在理解代码语法和语义上明显优于通用模型。总体而言，各类方法均有效提高了生成代码的功能正确度（通过测试）和质量，但在覆盖率、效率与安全性方面仍存在共同挑战。

# 趋势与挑战  
展望2025年前后，AI辅助代码生成领域可能出现以下趋势：  
- **多模态与上下文融合：** 未来研究将更多地结合软件工程相关信息（如文档、单元测试、运行时日志以及IDE上下文）进行生成。利用静态分析、运行时反馈或多模态输入（如问题描述+伪代码+流程图）将是热点，以进一步提升生成代码的语义一致性和正确性。  
- **交互式与人机协作：** 开发更加智能的交互式编程助手，使得用户与AI可以多轮对话逐步澄清需求和修正代码错误。研究者可能探索将LLM嵌入IDE、构建协同编程流水线，以及设计友好的反馈环，增强人类开发者对AI生成代码的可控性。  
- **验证与可靠性：** 随着生成模型广泛应用，对代码安全性、正确性和可维护性的要求会越来越高。未来工作会更多结合形式化验证、可解释性技术以及强化学习策略，使AI生成的代码符合法规与最佳实践，减少漏洞和偏差。  
- **模型效率与开放生态：** 尽管规模化趋势明显，但模型效率和部署成本需求促使研究包括模型蒸馏、压缩，以及针对特定任务/领域的精简模型开发。此外，开源代码模型和大规模公共基准（如HumanEval-X）会加速社区创新与透明度。  

# 结论  
本综述总结了近年AI辅助代码生成的主要技术路线，从大型预训练模型到检索增强、提示优化和迭代生成等多方面进行了分析。现有研究证明，结合领域知识（如程序语义、示例代码、测试反馈）和模型能力可以显著提高生成正确率和实用性。未来的工作将进一步强调生成结果的功能正确性、解释性以及与开发流程的紧密集成，推动AI编程助手向更可靠、高效的方向发展。
