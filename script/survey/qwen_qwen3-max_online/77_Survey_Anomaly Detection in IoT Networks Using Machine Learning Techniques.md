# 基于机器学习的物联网网络异常检测研究综述（2022–2025）

## 引言

物联网（IoT）设备的爆炸式增长在提升社会智能化水平的同时，也因其资源受限、安全机制薄弱及协议异构等特点，成为网络攻击的主要目标。传统的基于规则或签名的入侵检测系统难以应对针对物联网环境的未知、零日及高级持续性威胁（APT）。近年来，机器学习（ML）技术凭借其强大的模式识别与泛化能力，成为解决物联网网络异常检测（Anomaly Detection）问题的核心手段。本文聚焦2022至2025年间在顶级会议与期刊上发表的代表性工作，系统梳理了基于深度学习、小样本/零样本学习、联邦学习以及关系建模等前沿技术的异常检测方法，并总结其共性实验结论，进而预测未来研究趋势与核心挑战。

## 方法分类与代表作

### 1. 基于深度时空特征融合的检测方法

针对物联网流量复杂的时空依赖特性，研究者普遍采用深度神经网络进行联合建模。顾伟等人（2024）[jeit.ac.cn](https://jeit.ac.cn/cn/article/doi/10.11999/JEIT230825) 提出MSECNN-BiLSTM模型，利用一维CNN提取流量的空间特征，并引入多头挤压激励（MSE）机制进行自适应特征加权，再通过BiLSTM捕获长期时序依赖。在NSL-KDD数据集上，该模型F1-score达到90.13%，显著优于单一CNN或BiLSTM。浙江大学团队（2025）[zjujournals.com](https://www.zjujournals.com/eng/article/2025/1008-973X/20250611.shtml) 则设计了FL-CNN-LSTM框架，通过CNN与LSTM的并联结构分别提取时空特征，结合多头注意力机制筛选关键信息。其在CIC-IDS2018数据集上取得了99.00%的准确率，且并联结构相比传统串联结构性能增益显著。郑州工业应用技术学院团队（2025）[telecomsci.com](https://www.telecomsci.com/zh/article/doi/10.11959/j.issn.1000-0801.2025046) 提出了MRID模型，采用多尺度残差时间卷积网络（TCN）增强时空表征能力，并引入改进的流量注意力机制，在CICIDS2017和CICIDS2018上验证了其高效性与鲁棒性。

### 2. 面向类别不平衡与小样本场景的检测方法

物联网攻击流量天然具有稀疏性，导致训练数据严重不平衡。尹梓诺等人（2025）[jeit.ac.cn](https://jeit.ac.cn/cn/article/doi/10.11999/JEIT241115) 针对此问题，提出了一种无监督自适应抽样与改进孪生网络结合的方法。他们设计了基于K-medoids的自适应小样本抽样算法（KAFS），动态抽取更具代表性的均衡样本；并构建具有鲁棒损失函数的孪生多层感知机（SMLP）模型，学习流量间的异同。该方法在CICIDS2017上对未知攻击的检出率达99.61%，比现有方法至少高2.85%。

### 3. 基于联邦学习的隐私保护检测方法

为解决集中式训练带来的数据隐私泄露和数据孤岛问题，联邦学习（FL）被引入物联网异常检测。浙江大学团队（2025）[zjujournals.com](https://www.zjujournals.com/eng/article/2025/1008-973X/20250611.shtml) 的FL-CNN-LSTM框架允许各客户端在本地训练模型，仅上传模型参数至服务器进行聚合。实验表明，其联邦版本在CIC-IDS2018上的准确率（99.00%）与集中式版本（99.08%）性能相当，有效保护了数据隐私。此外，针对大模型训练集群的网络异常，刘贺与王晓湘（2025）[paper.edu.cn](https://www.paper.edu.cn/releasepaper/content/202504-3) 提出了一种流量与日志感知的实时检测框架，结合Transformer进行流量预测和基于日志的异常分析，能有效识别慢节点、链路延迟等影响训练效率的问题。

### 4. 基于关系挖掘与时序建模的检测方法

多变量时间序列（如服务器监控指标）的异常检测需考虑各变量间的隐式关系。胡智超等人（2024）[jcs.iie.ac.cn](https://jcs.iie.ac.cn/xxaqxb/ch/reader/view_abstract.aspx?flag=1&file_no=20240607&journal_id=xxaqxb) 提出TSAN算法，首先通过端到端的序列关系挖掘方法，结合节点嵌入相似性和因果推断构建关系图；再设计时空注意力网络进行联合预测。其在MSL和SMD数据集上的F1值分别优于次优方法0.9%和2.3%。孙怡阳等人（2024）[c-s-a.org.cn](https://www.c-s-a.org.cn/html/2024/1/9347.html) 则受MLP-Mixer启发，提出UMTS-Mixer模型，利用线性层（MLP）对时间维度和通道（变量）维度分别进行建模，充分利用了MLP对顺序的敏感性。该模型在MSL和PSM数据集上F1值分别达到91.35%和92.93%，验证了线性结构在时序建模中的有效性。

## 实验与评价总结

对上述工作的实验分析表明，现代ML驱动的IoT异常检测研究呈现出以下共性结论：（1）**数据集选择趋于标准化**：CIC-IDS2017/2018、NSL-KDD、UNSW-NB15及NASA的SMD/MSL/SMAP等已成为主流基准，便于公平比较。（2）**性能评价指标全面化**：除准确率外，研究者普遍采用精确率（Precision）、召回率（Recall）和F1-score等指标，尤其关注对少数类（攻击/异常）的召回率，以衡量模型对未知威胁的发现能力。（3）**特征工程向端到端学习演进**：从早期依赖手工特征提取，转向利用深度网络（如CNN、LSTM、Transformer、MLP）自动学习高阶、抽象的时空特征表示。（4）**模型复杂度与效率的权衡**：在追求高精度的同时，研究者通过模型压缩（如孪生网络）、高效架构（如TCN、MLP-Mixer）和联邦学习等手段，兼顾了模型在资源受限IoT环境中的部署可行性。

## 趋势与挑战

基于对近期文献的分析，2025年及以后的研究将聚焦于以下方向：（1）**大模型与异常检测的融合**：利用预训练大语言模型（LLM）或视觉-语言模型（VLM）对网络流量、日志、数据包载荷等多模态数据进行统一语义理解，实现更具泛化能力的异常检测，如LogFormer等工作的出现预示了这一趋势。（2）**因果推理与可解释性增强**：当前多数模型仍是“黑盒”，未来研究将深入结合因果推理技术（如反事实分析、去混淆学习），明确异常的根本原因，提升模型的可解释性和决策可信度，以满足安全运维的深度分析需求。（3）**动态环境下的持续学习**：物联网环境动态多变，设备行为、网络协议和攻击手法持续演进。如何构建能够在线、增量学习并有效抵御灾难性遗忘的持续学习（Continual Learning）或元学习（Meta-Learning）框架，是保障检测系统长期有效性的关键挑战。

## 结论

2022至2025年间，基于机器学习的物联网网络异常检测研究在模型架构、数据处理、隐私保护和关系建模等方面取得了显著进展。深度时空特征融合、小样本学习、联邦学习和关系挖掘成为四大主流技术路径。未来的研究将更加注重模型的泛化能力、可解释性及在动态环境中的持续适应能力，以应对日益复杂的物联网安全威胁。
