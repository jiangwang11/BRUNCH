### **基于大语言模型的推荐系统集成研究综述 (2022–2025)**

#### **引言**
推荐系统（Recommender Systems, RS）是信息过滤和个性化服务的核心技术，旨在解决信息过载问题。传统推荐模型，如协同过滤和基于内容的模型，在捕捉用户-物品交互的复杂模式方面取得了显著成就，但仍面临数据稀疏性、冷启动、以及对项目内容语义理解不足等固有挑战 [blog.csdn.net](https://blog.csdn.net/u013524655/article/details/147543857)。近年来，以大型语言模型（Large Language Models, LLMs）为代表的基础模型（Foundation Models, FMs）凭借其强大的自然语言理解、知识推理和生成能力，为推荐系统领域带来了颠覆性的变革。

本综述旨在系统性地梳理 2022 年至 2025 年间，将大型语言模型集成于推荐系统的代表性研究工作。我们将当前主流方法归纳为特征增强、生成式推荐和代理式推荐三种核心范式，并对各范式的代表性论文进行剖析，总结其关键实验发现，最后展望该领域面临的核心挑战与未来研究趋势。

#### **方法分类与代表作**

根据大型语言模型在推荐流程中扮演的角色，可将现有集成方法划分为三大范式：特征增强范式、生成式推荐范式和代理式推荐范式 [blog.csdn.net](https://blog.csdn.net/u013524655/article/details/147543857)。

##### **1. 特征增强范式 (Feature Enhancement Paradigm)**
该范式将 LLM 主要作为特征提取器或数据增强器，利用其世界知识和语义理解能力为传统推荐模型提供高质量的表示或补充数据，以缓解数据稀疏和语义理解不足的问题。

*   **LLMRec (2023)** [[hub.baai.ac.cn](https://hub.baai.ac.cn/paper/c2e79d28-5734-44af-925f-59b307fa2009)]
    *   **研究问题**: 如何解决推荐系统中的数据稀疏性问题，并克服辅助信息引入的噪声。
    *   **核心方法**: 提出一种基于 LLM 的图增强策略。具体而言，利用 LLM 对用户-物品交互图进行增强，包括强化交互边、增强物品节点属性理解和分析用户节点画像。同时，设计了一种去噪机制以保证增强数据的质量。
    *   **关键结论**: 实验表明，通过 LLM 进行图数据增强，能够有效利用内容信息来弥补隐式反馈的稀疏性，性能优于当时已有的技术。

*   **Large-TR (2025)** [[crad.ict.ac.cn](https://crad.ict.ac.cn/article/doi/10.7544/issn1000-1239.202440433?viewType=HTML)]
    *   **研究问题**: 在多模态推荐场景中，用户生成的内容（如评价、图片）中夹杂的噪声会限制推荐性能。
    *   **核心方法**: 提出一种基于 LLM 的可信多模态推荐算法。该算法利用 LLM 卓越的自然语言理解能力，高效地过滤多模态数据中的噪声。此外，设计了一种可信决策机制，用于动态评估推荐结果的不确定性，以提高高风险场景下的可用性。
    *   **关键结论**: 在四个公开数据集上的实验显示，该算法通过噪声过滤和不确定性评估，相较于基线算法取得了更优的性能表现。

*   **ELMRec (2024)** [[www.zhuanzhi.ai](https://www.zhuanzhi.ai/paper/166fc4c8042cf427064f1b766f21f2d2)]
    *   **研究问题**: 现有的基于 LLM 的推荐方法未能有效建模用户-物品之间的高阶交互信息，并且存在对用户早期交互而非近期交互的偏好问题。
    *   **核心方法**: 提出一种增强的 LLM 推荐模型 (ELMRec)，通过增强全词嵌入 (whole-word embeddings) 来提升 LLM 对图结构交互的理解能力，且无需进行图预训练。同时，引入一种重排序方案以纠正模型的近时性偏差。
    *   **关键结论**: 该方法在直接推荐和序列推荐任务中均超越了当时的 SOTA 方法，验证了在不进行图预训练的情况下，通过词嵌入增强也能有效整合图结构信息。

##### **2. 生成式推荐范式 (Generative Recommendation Paradigm)**
该范式将推荐任务重新定义为一个生成任务，直接利用 LLM 从庞大的物品库中生成推荐项目列表或相关解释，从而有望将传统的多阶段级联推荐简化为单阶段的端到端生成 [blog.csdn.net](https://blog.csdn.net/star_nwe/article/details/144927331)。

*   **Chat-REC (2023)** [[www.zhuanzhi.ai](https://www.zhuanzhi.ai/paper/71e5a34c2f88c9c3d7f816617416b8be)]
    *   **研究问题**: 传统推荐系统普遍存在交互性和可解释性差的局限。
    *   **核心方法**: 提出一个名为 Chat-Rec 的框架，通过将用户画像和历史交互行为转化为提示（Prompts），利用 ChatGPT 的上下文学习（In-context Learning）能力来理解用户偏好并进行对话式推荐。
    *   **关键结论**: 实验证明，该框架在 Top-K 推荐和零样本评分预测任务上表现出色，显著提升了推荐过程的交互性和可解释性，并能有效处理冷启动和跨领域场景。

*   **ETEGRec (2025)** [[blog.csdn.net](https://blog.csdn.net/weixin_46351593/article/details/147838045)]
    *   **研究问题**: 生成式推荐大多采用两阶段范式：先训练物品标记器（Item Tokenizer），再训练生成模型，两个阶段分离导致次优解。
    *   **核心方法**: 提出一个端到端的可学习生成式推荐框架 (ETEGRec)。该框架通过引入“序列-物品对齐”和“偏好-语义对齐”两个机制，协同优化 RQ-VAE 标记器和基于 Transformer 的生成器，使物品标记化过程和推荐生成过程能够联合训练。
    *   **关键结论**: 通过端到端联合优化，模型能够学习到更适合生成任务的物品表示，从而在生成式推荐任务上取得更优异的性能。

##### **3. 代理式推荐范式 (Agentic Recommendation Paradigm)**
这是最新的范式，将 LLM 塑造成一个具备规划、记忆和工具使用能力的自主代理（Agent）。该代理可以作为推荐系统本身，通过与用户多轮交互和动态推理来优化推荐；也可以作为用户模拟器，为评估和训练推荐系统生成高质量的模拟行为数据 [blog.csdn.net](https://blog.csdn.net/u013524655/article/details/147543857)。

*   **Agent4Rec (2023)** [[blog.csdn.net](https://blog.csdn.net/u013524655/article/details/147543857)]
    *   **研究问题**: 高质量的用户行为数据收集成本高昂且在伦理上复杂，难以用于全面评估和优化推荐系统。
    *   **核心方法**: 构建了一个拥有上千个由 LLM 驱动的生成代理的推荐系统模拟器，这些代理能够模拟真实人类在推荐场景下的行为（如浏览、点击）。
    *   **关键结论**: 研究发现，LLM 赋能的生成代理能够准确模拟真实人类在电影推荐等场景中的行为模式，为离线评估和算法迭代提供了有效途径。

*   **InteRecAgent (2023)** [[blog.csdn.net](https://blog.csdn.net/u013524655/article/details/147543857)]
    *   **研究问题**: 如何让 LLM 在推荐任务中有效利用领域知识并执行多步推理。
    *   **核心方法**: 将 LLM 视作代理的大脑，并为其配备一系列推荐专用工具（如信息查询、项目检索、项目排序）。代理可以解析用户意图，自主调用这些工具来完成复杂的推荐任务。
    *   **关键结论**: 通过工具使用，代理式推荐系统能够将 LLM 的通用推理能力与推荐模型的领域知识相结合，实现更灵活和精准的交互式推荐。

#### **实验与评价总结**
综合分析 2022-2025 年间的相关研究，可以总结出以下共性实验结论：

1.  **性能提升的普遍性**: 在多个公开数据集（如 Amazon、Yelp、MovieLens）上，与传统推荐模型（如 SASRec）相比，集成了 LLM 的方法在 NDCG、Recall 等主流排序指标上普遍表现出优势，尤其是在利用了丰富的文本或多模态信息时。
2.  **效率成为关键瓶颈**: 尽管性能优越，但 LLM 的巨大计算开销是其在工业界大规模部署的核心障碍。如 **SLMRec (2025)** [[blog.csdn.net](https://blog.csdn.net/qq_35812205/article/details/146285120)] 的研究明确指出了这一点，并通过知识蒸馏技术，在仅使用 13% 参数量的情况下，实现了与大型模型相媲美的性能，同时训练和推理速度分别提升 6.6 倍和 8.0 倍。这表明模型小型化是未来的重要方向。
3.  **生成式推荐的“幻觉”问题**: 直接生成推荐结果的范式面临着推荐出数据库中不存在的物品（即“幻觉”）的风险，这会严重损害用户体验。为此，研究者们通过约束解码空间（如基于Trie树的搜索）或与现有物品库进行对齐来缓解此问题 [blog.csdn.net](https://blog.csdn.net/star_nwe/article/details/144927331)。
4.  **评估体系的局限性**: 传统推荐指标无法完全衡量 LLM 带来的新价值，如解释的合理性、对话的流畅性以及交互的满意度。未来的评估体系需要更加多元化，结合人工评估和在线 A/B 测试来综合评判模型效果。

#### **趋势与挑战**
基于当前研究进展，预计 2025 年前后的研究将聚焦于以下趋势与挑战：

1.  **模型小型化与效率优化**: 随着 SLMRec 等工作的出现，如何通过知识蒸馏、模型剪枝、量化等技术，在保持 LLM 带来的语义理解和推理优势的同时，大幅降低模型参数量和推理延迟，将成为研究热点。这标志着研究重心从“如何用大模型”转向“如何高效地用模型”。
2.  **端到端生成与表示对齐**: 以 ETEGRec 为代表的端到端框架显示了联合优化物品表示和生成过程的巨大潜力。未来的研究将进一步探索更优的对齐技术（如对比学习、最优传输），以弥合协同信息、语义信息和 LLM 生成偏好之间的鸿沟，实现真正无缝的端到端生成式推荐。
3.  **多模态代理式智能的融合**: 结合多模态理解能力和代理式框架是推荐系统迈向更高阶智能的必然路径。未来的研究将致力于构建能够理解并响应多模态输入（如图片、语音）的推荐代理，它们不仅能与用户进行自然对话，还能利用外部工具（如搜索引擎、知识库）进行主动信息探寻和多步规划，以满足用户更复杂、动态和情境化的需求。
4.  **可信赖与可控性**: 随着 LLM 在推荐中扮演更核心的角色，其带来的幻觉、偏见和隐私泄露等问题愈发突出。如何设计更具可解释性、公平性和可控性的 LLM 推荐系统，例如通过将 LLM 的决策过程与知识图谱中的显式路径对齐，或在代理框架中引入伦理原则约束，是确保技术健康发展的关键挑战 [blog.csdn.net](https://blog.csdn.net/star_nwe/article/details/144927331)。

#### **结论**
大型语言模型（LLM）的集成正在深刻地重塑推荐系统的研究范式。本综述回顾了 2022 年至 2025 年的代表性工作，将现有方法归纳为特征增强、生成式和代理式三大范式。研究表明，LLM 不仅能作为强大的特征提取器来缓解传统推荐模型的局限，更能以生成者或自主代理的角色，开启端到端、可解释、交互式的推荐新模式。尽管在模型效率、幻觉问题和评估方法上仍面临挑战，但模型小型化、端到端对齐以及多模态代理智能的融合已成为清晰可见的研究趋势。未来，构建更高效、智能、可信的 LLM 驱动的推荐系统将是该领域持续探索的核心方向。
