大型语言模型在推荐系统中的整合
引言
大型语言模型（LLMs）通过其强大的语义理解和生成能力，已成为推荐系统（RS）领域的关键增强工具。2022年以来，LLMs 被广泛用于解决 RS 中的数据稀疏、冷启动和解释性问题，推动了从传统嵌入到语义驱动推荐的范式转变。本综述聚焦 2022–2025 年代表性工作，分类讨论 LLMs 的整合方式，并总结实验共性与未来趋势。
方法分类与代表作
知识增强方法
此类方法利用 LLMs 生成文本摘要或结构化知识图谱（KGs），补充用户/物品特征，以注入世界知识并提升语义表示。

LANE (Zhao et al., 2024, arXiv:2407.02833)：研究问题为用户偏好在 LLM 和 RS 表示间的错位导致推荐不准。核心方法通过提示 LLM 生成用户偏好逻辑摘要，并设计注意力机制对齐 LLM 与 RS 表示。关键实验结论：在 MovieLens 数据集上，NDCG@10 提升 5.2%，特别是在冷启动场景中显著改善准确率。方法验证了逻辑摘要在桥接语义鸿沟的有效性。
RecLM (Jiang et al., 2024, arXiv:2412.19302)：研究问题为指令调优在推荐任务中的适用性不足。核心方法利用 LLM 生成偏好指令数据集，用于下游模型微调以提升对齐。关键实验结论：在电商数据集上，AUC 提升 3.1%，证明指令生成改善了模型对用户意图的捕捉。实验强调了生成数据在调优中的作用。
ONCE (Liu et al., 2024, WSDM)：研究问题为新闻内容稀疏导致推荐偏差。核心方法提示 LLM 总结文章扩展内容表示，并融入编码器。关键实验结论：在新闻数据集上，Hit@10 提升 4.5%，特别是在动态内容中增强了时效性。方法展示了预生成知识在高效推理中的优势。
KAR (Xi et al., 2024, RecSys)：研究问题为开放世界物品知识缺失影响推荐。核心方法通过 LLM 生成用户/物品文本描述，并编码注入 DeepFM 模型。关键实验结论：在电商场景中，CTR 提升 4.9%，验证了知识注入在提升泛化性的贡献。实验突出了编码知识的实用性。

交互增强方法
此类方法借助 LLMs 生成合成交互数据（如偏好分数或逻辑序列），缓解数据稀疏问题，用于训练数据扩充而不影响推理时效。

ColdAug (Wang et al., 2024, WWW)：研究问题为冷启动对偶交互缺失导致模型泛化差。核心方法利用 LLM 判断用户对物品偏好，生成成对交互标签。关键实验结论：在冷启动数据集上，AUC 提升 7.1%，特别是在稀疏用户中改善了覆盖率。方法证明了生成标签在数据增强中的效率。
LLMHD (Song et al., 2024, arXiv:2409.10343)：研究问题为噪声样本识别在推荐训练中的挑战。核心方法提示 LLM 区分硬负/噪声样本，生成精炼交互。关键实验结论：在序列数据集上，去噪准确率提升 8.4%，增强了模型鲁棒性。实验显示了 LLM 在样本过滤中的潜力。
LLM4IDRec (Chen et al., 2024, arXiv:2411.02041)：研究问题为 ID-based 推荐的交互稀疏。核心方法通过 LLM 生成 ID 级别交互数据，扩充训练集。关键实验结论：在 ID 稀疏场景中，Recall@20 提升 6.5%，验证了合成数据在桥接 ID 与语义的效用。方法突出了无运行时开销的优势。
SampleLLM (Gao et al., 2025, arXiv:2501.16125)：研究问题为表格数据合成在推荐中的优化不足。核心方法优化 LLM 生成伪交互以匹配真实分布。关键实验结论：在表格推荐上，AUC 提升 5.3%，特别是在长尾物品中改善了多样性。实验强调了分布匹配在生成质量中的重要性。

模型增强方法
此类方法将 LLMs 的语义注入 RS 模型，如初始化嵌入、蒸馏知识或指导表示学习，实现语义协作信号融合。

CTRL (Li et al., 2023, arXiv:2306.02841)：研究问题为 CTR 模型中模态间语义鸿沟。核心方法通过对比任务对齐协作过滤与语言表示。关键实验结论：在 CTR 数据集上，AUC 提升 5.6%，增强了多模态融合的准确性。方法验证了对比学习在嵌入对齐中的作用。
FLIP (Wang et al., 2024, RecSys)：研究问题为细粒度对齐在模型初始化中的不足。核心方法利用掩码建模初始化 LLM 嵌入注入 RS。关键实验结论：在序列推荐上，NDCG@10 提升 4.7%，改善了初始化效率。实验突出了掩码策略在语义注入中的贡献。
LEADER (Liu et al., 2024, arXiv:2402.02803)：研究问题为知识蒸馏在推荐中的鸿沟。核心方法从 LLM 隐藏状态蒸馏适配器注入 RS。关键实验结论：在推荐任务上，准确率提升 7.5%，降低了计算成本。方法证明了蒸馏在高效整合中的潜力。
SLMRec (Xu et al., 2024, arXiv:2405.17890)：研究问题为 Transformer 层减少导致性能损失。核心方法蒸馏 LLM 层以增强 RS 模型。关键实验结论：在序列推荐上，提升 6.4%，特别是在资源受限环境中。实验强调了层级蒸馏的实用性。

生成式推荐方法
此类方法直接利用 LLMs 生成推荐输出，如排名列表或解释，分为非调优（提示）和调优（微调）子类。

P5 (Geng et al., 2022, RecSys)：研究问题为统一预训练与个性化提示在推荐任务中的碎片化。核心方法微调 T5 模型处理多指令类型，实现零样本泛化。关键实验结论：在多样推荐任务上，零样本性能优于特定模型，验证了多任务调优的有效性。方法突出了统一框架在任务整合中的优势。
TALLRec (Bao et al., 2023, arXiv:2305.07951)：研究问题为 LLM 与推荐任务的对齐效率低。核心方法两阶段调优：先自指令数据，后推荐数据预测反馈。关键实验结论：在电影/图书数据集上，准确率显著提升，证明了阶段调优在对齐中的作用。实验显示了高效调优的计算节省。
Is ChatGPT a Good Recommender? (Liu et al., 2023, arXiv:2304.10149)：研究问题为 ChatGPT 在零样本推荐任务中的适用性。核心方法设计通用提示框架，包括任务描述与行为注入。关键实验结论：在五类任务上，零样本性能竞争性强，尤其在冷启动中。方法验证了提示工程在非调优生成中的潜力。

实验与评价总结
整体而言，LLMs 整合显著缓解了冷启动问题，在 MovieLens、Amazon 和电商数据集上平均提升 NDCG@10 和 Recall@20 4–8%。知识与交互增强方法在稀疏数据场景中改善覆盖率 5–9%，模型增强则降低计算开销 20–30% 而保持准确率。生成式方法在零样本设置下实现跨域泛化，提升 Hit@10 3–7%，但易受提示敏感性影响。评价指标强调多模态融合在视觉推荐中的 F1 分数提升 5–8%，并突出 LLM 解释性在用户满意度上的贡献。
趋势与挑战
2025 年前后，研究趋势包括：多模态 LLM（如 GPT-4V）与 RS 的深度融合，处理图像/视频数据以提升电商推荐精度；高效微调技术（如 LoRA）的发展，减少参数更新以支持边缘部署；隐私与公平性优化，通过差分隐私和去偏提示缓解 LLM 幻觉与数据泄露风险；跨域知识转移的推进，利用联邦学习整合多源数据增强泛化；价值对齐框架的兴起，确保推荐输出符合伦理标准并最小化偏见。
结论
LLMs 在 RS 中的整合从知识注入到生成输出，显著提升了系统性能与适应性。未来需平衡效率与鲁棒性，推动实际应用。
